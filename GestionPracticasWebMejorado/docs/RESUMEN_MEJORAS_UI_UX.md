# Resumen de mejoras UI/UX aplicadas

## Director

- Se mantiene como rol de supervisión estratégica.
- Dashboard con indicadores de usuarios, prácticas, instituciones y horas.
- Módulos de usuarios, prácticas, cursos, instituciones y reportes.
- Usuarios con activar, inactivar, aprobar, editar y eliminar.
- Se eliminó el texto “Oracle 10g” del inicio visual y se dejó “Panel administrativo”.

## Coordinador

- Se eliminó la bienvenida personalizada con nombre específico y se dejó “Bienvenido Coordinador”.
- Se reforzó como segundo responsable operativo del sistema.
- CRUD de grupos de práctica.
- CRUD de asignación docente.
- Control de estudiantes, instituciones y horas.
- Gestión orientada a preinscripción, plan de trabajo, evaluación de informes y relación con instituciones.

## Estudiante

- Dashboard con bienvenida, paso a paso y notificaciones pendientes.
- Carga de foto de perfil.
- Módulos de matrícula, documentos, actividades, notas, promedio e inquietudes.
- Mensaje claro cuando no existe matrícula aprobada.
- Se incluyó compatibilidad con instituciones mediante porcentajes.
- Se organizó la navegación para que el estudiante sepa qué debe hacer primero.

## Docente

- Dashboard de revisión de actividades.
- Revisión de bitácoras, horas y comentarios.
- Rúbricas, preguntas, evaluaciones y retroalimentación.
- Respuestas rápidas para evitar editores pesados y reducir consumo de recursos.

## Institución receptora

- Dashboard con convenio, practicantes, horas y certificación.
- Se incluyó flujo por ingresar, activos y finalizados.
- Se agregó vista de convenio, supervisor, evaluación final y certificación.
- Se añadió compatibilidad estudiante-institución.
- Se habilita logo institucional en perfil.

## General

- Todas las pantallas usan organización por estados.
- Todas las UI principales tienen estructura CRUD.
- Se agregó chat interno tipo correo privado.
- Perfil editable con foto/logo, comentario y cambio de contraseña.
- Diseño responsive para funcionar en equipos de bajos recursos.
- Se evita cargar toda la información detallada hasta que el usuario selecciona “Ver”.
