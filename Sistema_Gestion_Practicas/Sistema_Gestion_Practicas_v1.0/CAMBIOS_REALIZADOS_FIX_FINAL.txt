PROYECTOJD - Cambios realizados en el ZIP corregido
====================================================

1. Dashboard Director
- Se quitó el texto "Oracle 10g" del encabezado.
- Ahora el subtítulo queda como: "Panel administrativo".

2. CRUD genérico del sistema
- Se mejoró CrudTablaFrame para que todos los módulos genéricos tengan:
  * Guardar / Actualizar.
  * Nuevo.
  * Activar.
  * Inactivar.
  * Eliminar definitivo.
  * Refrescar.
- La tabla ahora ordena por estados de proceso:
  ACTIVO, PENDIENTE, APROBADO, REGISTRADO/REGISTRADA, ENVIADA, RECHAZADO/REPROBADO, FINALIZADO/FINALIZADA, INACTIVO, ELIMINADO.
- Se agregó conversión básica de datos para columnas NUMBER y DATE al guardar desde los CRUD genéricos.

3. Dashboard Coordinador
- Se quitó el nombre del usuario en el encabezado y quedó: "Bienvenido Coordinador".
- Se retiró el botón "Crear Datos Base" porque el sistema debe asumir conexión real a la base de datos.
- Grupos de práctica ahora tiene CRUD real con la tabla GRUPO.
- Asignación Docente abre CRUD completo conectado a ASIGNACION_DOCENTE.

4. Dashboard Docente
- Se retiró el botón "Crear Datos Base".
- Se mantiene revisión de actividades, rúbricas, preguntas, evaluaciones y retroalimentación conectadas a tablas reales.
- Los CRUD genéricos ahora permiten activar, inactivar y eliminar definitivo cuando aplique.

5. Dashboard Estudiante
- Se retiró el botón "Crear Matrícula Prueba".
- Se agregó el botón "Mi matrícula / proceso" para consultar el proceso real de matrícula.
- Se ajustó el texto del flujo: el estudiante debe solicitar asignación al Coordinador o Docente antes de registrar actividades si no tiene matrícula.
- Se agregó una tarjeta de paso a paso del estudiante.

6. Dashboard Institución Receptora
- Se retiró el botón "Crear Institución Base".
- Se mantuvo la consulta de estudiantes, prácticas, datos de institución y confirmación de horas.
- Se agregó soporte de columna LOGO si la base de datos la tiene.

7. Usuarios
- En Gestión de Usuarios se agregó:
  * Activar usuario.
  * Inactivar usuario.
  * Eliminar definitivo.
- El mensaje superior de ayuda se hizo más visible.
- Los usuarios se ordenan por estado.

8. Cursos
- En Gestión de Cursos se agregó:
  * Activar Curso.
  * Inactivar Curso.
- Se mantiene Guardar, Actualizar y búsqueda.
- Los cursos se ordenan por estado.

9. Script de base de datos agregado
- Se creó el archivo database/08_actualizacion_ui_estados_perfil.sql.
- Este script es OPCIONAL y se debe ejecutar solo si se desea agregar:
  * USUARIO.FOTO_PERFIL.
  * INSTITUCION.LOGO.
  * REGISTRO_ACTIVIDAD.REVISADO_POR.
  * REGISTRO_ACTIVIDAD.FECHA_REVISION.
  * Estados ampliados para INSTITUCION: ACTIVO, INACTIVO, PENDIENTE, APROBADO, REPROBADO.

10. Validación técnica
- Se ejecutó compilación con Ant.
- Resultado: BUILD SUCCESSFUL.
- Quedan advertencias por constructores antiguos Integer/Double, pero no impiden ejecutar el programa.

IMPORTANTE PARA LA BASE DE DATOS
====================================================
Los cambios principales del programa NO obligan a cambiar las tablas actuales.
Solo hay un cambio opcional de base de datos:

Ejecutar database/08_actualizacion_ui_estados_perfil.sql si vas a usar:
- Foto de perfil del usuario.
- Logo de institución receptora.
- Estados PENDIENTE/APROBADO/REPROBADO en INSTITUCION.
- Historial simple de revisión de actividades.

Si tu base actual tiene restricción CHECK en INSTITUCION.ESTADO solo para ACTIVO/INACTIVO, debes ejecutar ese script antes de usar PENDIENTE, APROBADO o REPROBADO en instituciones.
