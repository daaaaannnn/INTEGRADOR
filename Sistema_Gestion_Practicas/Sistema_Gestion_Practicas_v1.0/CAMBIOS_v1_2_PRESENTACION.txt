Sistema_Gestion_Practicas_v1.2_presentacion

Cambios aplicados para la presentación:

1. Se corrigieron consultas incompatibles con la base Oracle existente.
   - La interfaz ya no asume que EVALUACION tenga NOTA_EVALUACION.
   - Si la base solo tiene PUNTAJE_OBTENIDO, CALIFICACION, NOTA o NOTA_FINAL, el sistema usa la columna disponible.
   - PRACTICA se consulta por TITULO, NOMBRE o NOMBRE_PRACTICA según exista.
   - PREGUNTA se consulta por ENUNCIADO o TEXTO_PREGUNTA según exista.
   - RESPUESTA se consulta por CONTENIDO_RESPUESTA o RESPUESTA según exista.

2. Matrícula simplificada.
   - El coordinador selecciona estudiante, práctica, institución y grupo desde combos.
   - No se escriben ID manuales.
   - Si el estudiante ya está matriculado, la tabla se refresca para mostrarlo.
   - La tabla muestra estudiante, práctica, grupo, institución, etapa productiva, horas reportadas, horas aprobadas, horas pendientes y promedio.

3. Evaluaciones docente-estudiante.
   - El docente selecciona una matrícula real.
   - Publica preguntas para que el estudiante las vea en su cuestionario.
   - Se crea una rúbrica general automáticamente cuando el esquema exige ID_RUBRICA.
   - El docente puede calificar la evaluación seleccionada y la nota queda visible para el estudiante y para reportes.

4. Cuestionario del estudiante.
   - El estudiante ve las preguntas publicadas para sus matrículas.
   - Puede responder y actualizar su respuesta.
   - La pantalla funciona aunque RESPUESTA use ID_USUARIO o ID_ESTUDIANTE.

5. Control de horas.
   - El estudiante registra horas.
   - Docente, coordinador o institución pueden aprobar o rechazar.
   - La tabla muestra histórico con horas reportadas, aprobadas, pendientes, estado y observación.
   - Al aprobar se registra historial y bitácora.
   - Si las horas aprobadas alcanzan las requeridas por la práctica, se marca la matrícula como PRODUCTIVA_COMPLETADA cuando la columna existe.

6. Reportes e histórico.
   - Se agregó vista "Matrículas y avance productivo".
   - Se corrigieron reportes para no fallar cuando una columna no existe.
   - El administrador/director puede ver matrículas, horas, evaluaciones, respuestas, LOG_ACTIVIDAD, HISTORIAL_REVISION y reportes generados.

Compilación:
- Ejecutado: ant clean jar
- Resultado: BUILD SUCCESSFUL
